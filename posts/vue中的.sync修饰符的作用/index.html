Vue中的.sync修饰符的作用

January 11, 2020

.sync修饰符的作用

  • 直接来例子

    <div id="app">
<div>{{bar}}</div>
<my-comp :foo.sync="bar"></my-comp>
</div>

<script>
Vue.component('my-comp', {
    template: '<div @click="increment">点我+1</div>',
    props: ['foo'],
    methods: {
        increment: function() {
            this.$emit('update:foo', this.foo+1);
        }
    }
});
new Vue({
    el: '#app',
    data: {bar: 0}
});
</script>

    <my-comp :foo.sync="bar"></my-comp>  => 
等价于 <my-comp :foo="bar" @update:foo='bar=$event' ></my-comp>   
$event代表触发事件时抛出的值

总结

  • vue 修饰符sync的功能是：当一个子组件改变了一个 prop 的值时，这个变化也会同步到父组件中所绑定。如果我们不用.sync，我们想做上面的那个弹窗功能，我们也可以props传初始值，然后事件监听，实现起来也不算复杂。其实这个修饰符就是一个语法糖，但是我们需要明白其中的原理，才能更好的去使用和掌握他。
